# Quickstart: Docusaurus Setup for ROS 2 Educational Content

## Prerequisites

- Node.js v18 or higher
- npm or yarn package manager
- Basic knowledge of Markdown and Git

## Setup Instructions

### 1. Install Docusaurus

```bash
# Create a new Docusaurus project
npx create-docusaurus@latest website classic

# Navigate to project directory
cd website

# Install additional dependencies for diagrams
npm install @docusaurus/module-type-aliases
npm install @docusaurus/types
npm install @docusaurus/plugin-content-docs
```

### 3. Set up Sidebars

Create `sidebars.js`:

```javascript
// sidebars.js
/** @type {import('@docusaurus/plugin-content-docs').SidebarsConfig} */
const sidebars = {
  tutorialSidebar: [
    {
      type: 'autogenerated',
      dirName: 'modules',
    },
  ],
};

export default sidebars;
```

### 4. Create Module Directory Structure

```bash
mkdir -p docs/modules/001-ros2-robotic-nervous-system
mkdir -p docs/modules/001-ros2-robotic-nervous-system/chapter-1-introduction
mkdir -p docs/modules/001-ros2-robotic-nervous-system/chapter-2-communication
mkdir -p docs/modules/001-ros2-robotic-nervous-system/chapter-3-bridging
```

### 5. Create Your First Content File

Create `docs/modules/001-ros2-robotic-nervous-system/intro.md`:

```markdown
---
sidebar_position: 1
---

# Module 1: The Robotic Nervous System (ROS 2)

This module introduces ROS 2 as the middleware that connects perception, decision-making, and actuation in robots.

## Learning Objectives

- Understand ROS 2's role in Physical AI
- Identify core ROS 2 components
- Conceptually map ROS 2 to humanoid robot control
```

### 6. Run the Development Server

```bash
npm run start
```

This will start the development server at `http://localhost:3000`.

## Adding Content

1. Create Markdown files in the appropriate module/chapter directories
2. Add content using Docusaurus Markdown features
3. Update sidebars.js to control navigation order if needed
4. Use code blocks with syntax highlighting for Python examples
5. Use Mermaid diagrams for system architecture illustrations

## Deployment

For GitHub Pages deployment:

1. Update the `url`, `baseUrl`, `organizationName`, and `projectName` in `docusaurus.config.js`
2. Run: `npm run build`
3. The static site will be in the `build/` directory
4. Configure GitHub Pages to serve from the `gh-pages` branch
